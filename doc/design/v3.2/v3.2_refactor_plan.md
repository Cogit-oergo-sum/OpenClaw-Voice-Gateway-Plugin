# Fast Agent V3.2 架构重构与模块化计划

## 1. 整体方案 (Overall Architecture Plan)

V3.2 版本的核心目标是对日趋庞大的“上帝对象” `FastAgentV3` (700+行) 进行垂直职责切分，使其回归“调度器 (Orchestrator)”的本质。秉承奥卡姆剃刀原则，我们采用“重构但不改变核心业务逻辑 (Refactoring without changing external behavior)”的方式分离关注点。

### 1.1 模块拆分策略
*   **CanvasManager (`src/agent/canvas-manager.ts`)**: 负责管理核心画布 (Canvas) 的生命周期、内存同步 (`syncCanvasesFromDisk`) 及审计日志记录 (`logCanvasEvent`)，暴露状态读写的单点出口。
*   **DelegateExecutor (`src/agent/executor.ts`)**: 接管原有强耦合的 `openclaw agent ...` 子进程调用、`TIMEOUT_5S` 超时赛跑逻辑及命令行标准输出的正则匹配/提取。只对外返回干净的 JSON 解析结果或执行错误文本。
*   **WatchdogService (`src/agent/watchdog.ts`)**: 将原先混杂在 Agent 内部的定时器心跳剥离，重构为基于事件回调的监听模式。当扫描到 Canvas `status: READY` 时，触发事件给调度器，彻底避免 Watchdog 与生成逻辑强耦合。
*   **SLCEngine & SLEEngine (`src/agent/slc.ts`, `src/agent/sle.ts`)**: 隔离两路的 LLM 推理引擎及 Prompt 组装。SLC 专注垫词拼接与交互层潜意识注入，SLE 专注执行后的任务总结。
*   **TextCleaner (`src/utils/text-cleaner.ts`)**: 原本散落在各处的正则脱敏逻辑 (`decant`, `cleanText`) 收拢为全局防噪音的无状态纯函数工具类。

### 1.2 重构后的系统交互链路
```mermaid
graph TD
    Facade[FastAgentV3 Facade] -->|1. process()| SLC[SLCEngine]
    Facade -->|2. process()并发| SLE[SLEEngine]
    SLE -->|调工具| Executor[DelegateExecutor]
    SLE -->|总结写入| Canvas[CanvasManager]
    Watchdog[WatchdogService] -->|心跳扫描/审计监控| Canvas
    Watchdog -->|触发回调| Facade
    Facade -->|3. 内置触发| SLC
```

## 3. 测试用例 (Test Cases)

重构前后的外在表现必须保持绝对一致。请参考以下回归用例：

### 3.1 极速交互验证 (SLC)
- **用例 1 (瞬时应答)**: 用户说“你好”，系统在 600ms 内触发 `filler`（如“您好，先生”），响应不等待 SLE 推理完成。
- **用例 2 (潜意识缝合)**: 用户下发工具指令，SLC 先说出“稍等我看看喔”，事后结果回来，必须自然衔接“我看过了，文件已创建好”，中间不得出现语义断层或复读“稍等我看看喔”。

### 3.2 强逻辑校验与工具委派 (SLE & Executor)
- **用例 3 (工具调用成功)**: 命令“查看 checklist”。SLE 调用 OpenClaw 执行，返回结果，Canvas 状态置为 `READY`。通过 Butler 转化逻辑输出对先生有意义的总结反馈。
- **用例 4 (工具5s超时竞速)**: 请求长耗时指令“列出所有历史版本”。如果在 5s 内工具未返回，SLE 应当中断当前的工具生成，先行说出“这需要花点工夫，我替您盯着” (WaitingMsg)。
- **用例 5 (画布落盘审计)**: 验证所有的 `CANVAS_INIT`, `CANVAS_PENDING`, `CANVAS_CLI_READY` 事件严格正确地记录在 `workspace/logs/canvas.jsonl` 中，证明 `CanvasManager` 正常工作。

### 3.3 Watchdog 守护功能
- **用例 6 (内部主动触发)**: 模拟长耗时异步任务最终结束。Canvas 中数据转为 `READY`，Watchdog 扫描到该事件，主动触发 `__INTERNAL_TRIGGER__`，系统主动开口告知任务成果，不需要用户二次提问。 
- **用例 7 (防噪音与脱敏测试)**: 所有送至 TTS 或推给前端的回复，不得包含任何形如 `(潜意识思考: ...)`、`[调用 openclaw...]` 或 `HEARTBEAT_OK` 等内容代码，证明 `TextCleaner` 模块依然生效。
