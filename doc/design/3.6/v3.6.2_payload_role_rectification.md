# V3.6.2 Payload & Role Rectification - Alignment Plan

## 1. 现状回顾与问题定位 (Analysis)
基于对 V3.6.0/3.6.1 运行结果与 `v3.6-layered-logic-context.md` 的对比，发现当前实现存在以下偏离：

| 偏离项 | 现状 (Status Quo) | 需求 (Requirement) | 影响 (Impact) |
| :--- | :--- | :--- | :--- |
| **消息拓扑** | [画布快照] 揉杂在 User 历史中 | [画布快照] 做为独立外部输入注入 | 专家角色对环境感知的边界模糊 |
| **潜意识注入** | 注入了给 SLC 的垫词 (Waiting/Idle) | 仅保留 [Shadow Thought] 逻辑预判 | 逻辑专家产生冗余操作或决策迟钝 |
| **协议纯度** | `SLE_ACTION_PROTOCOL` 包含 TTS 规范 | 逻辑专家仅关注任务判定 | 稀释 AI 在决策时的注意力与 Token 浪费 |
| **输出确定性** | 文本自由输出，无 JSON 约束 | 严格 JSON 格式输出 | `SLEEngine` 对工具调用的解析存在幻觉风险 |

## 2. 核心修正原则 (Rectification Principles)
1.  **极简主义 (Minimalism)**：逻辑专家 (SLE) 不应感知任何语音播报规则。
2.  **原子隔离 (Atomic Isolation)**：画布状态 (Context) 是外部观察量，用户语音 (Input) 是即时变量，必须分层。
3.  **确定性闭环 (Deterministic Loop)**：决策输出必须是可解析的对象 (JSON Object)。

## 3. 架构演进 (Architecture Evolution)

### 3.1 场景 B (DECIDING) Payload 结构校准
```text
System: [逻辑专家人设] + [纯净行动协议] + [原子工具库]
User: [实时画布快照] + [Intent Hint]
History: [User/Assistant 交替历史]
User: [用户最新语音或触发词]
```

### 3.2 决策协议 JSON 模型
```json
{
  "thought": "对当前语境、历史和工具的逻辑分析",
  "intent": "当前锁定的任务意图",
  "response": "如果不调用工具，给出的理性的、无需润色的事实回答"
}
```

## 4. 实施路径 (Implementation Path)
针对复杂的逻辑校准，分为以下由独立 Agent 执行的开发阶段：

### 4.1 开发计划 (Development Phases)

| 阶段 | 核心任务 | 说明 | 验证手段 |
| :--- | :--- | :--- | :--- |
| **Phase 1: 协议脱敏与标准化** | 瘦身 `prompts.ts` 中的 `SLE_ACTION_PROTOCOL` | 剥开 `TTS_FRIENDLY_PROTOCOL`，注入强制 JSON 格式指令。确保逻辑专家与交互专家职责分离。 | `verify_v3.6.2_p1.ts` |
| **Phase 2: 消息分层与潜意识脱离** | 重构 `sle-payload-assembler.ts` 的 `DECIDING` 逻辑 | 将 [Canvas Snapshot] 作为独立 `user` 注入。完全移除 `buildShadowThought` (垫词) 补全，净化推理场景。 | `verify_v3.6.2_p2.ts` |
| **Phase 3: 引擎升级与 JSON 闭环** | 升级 `sle.ts` 引擎支持 `json_object` | 开启 JSON 响应模式。实现同步解析 `thought` / `intent` / `response`，并兼容标准 `tool_calls`。 | `verify_v3.6.2_p3.ts` |
| **Phase 4: 全量回归与文档同步** | 运行集成验证并更新 `overview.md` | 使用真实天气任务模拟 (ASR/Decision/Summarize)，确保链路逻辑严密且不符合点全部消除。 | `verify_v3.6.2_final.ts` |

## 5. Coding Agent 指令 (Subagent Tasks)

### 🤖 Coding Agent A: 协议标准化专家 (Phase 1)
**项目背景**：OpenClaw RTC V3.6.2 协议校准。
**核心参考**：`src/agent/prompts.ts`。
**任务详述**：
1.  **修改 `SLE_ACTION_PROTOCOL`**：
    -   移除 `${TTS_FRIENDLY_PROTOCOL}` (语音输出规范) 的引用。
    -   移除冗余的 `${LOGIC_EXPERT_IDENTITY}` 注入 (由 Assembler 外部注入)。
    -   新增 `# 输出格式 (Output Format)`：要求模型必须且仅能输出严格 JSON，包含 `thought` (思维链), `intent` (意图), `response` (直答文本) 三个字段。
2.  **强制闭环**：编写 `verify_v3.6.2_p1.ts`，导出该 Prompt 字符串，验证其中不含 "Voice Output Rules" 字样，且包含具体的 JSON 结构定义。脚本必须显式调用 `process.exit(0)`。

---

### 🤖 Coding Agent B: 消息分层处理专家 (Phase 2)
**项目背景**：解决 V3.6.1 中 Canvas 快照与对话历史揉杂的问题，并移除 SLC 垫词注入。
**核心参考**：`src/agent/sle-payload-assembler.ts`。
**任务详述**：
1.  **重构 `DECIDING` 场景组装逻辑**：
    -   **协议拆分**：首个 `user` 角色消息应仅包含 `snapshotStr` ([Canvas Snapshot] + [Intent Hint])。
    -   **历史独立**：随后 `push(...dialogueHistory)`，禁止将 Snapshot 强行 merged 进第一条历史 user 中。
    -   **净化场景**：删除 line 83-92 中关于 `buildShadowThought` 的 `assistant` 预填充逻辑。
2.  **强制闭环**：编写 `verify_v3.6.2_p2.ts`，模拟 `dialogueHistory.length > 0` 场景，验证生成的 `messages` 数组中，第一条 user 消息等于 snapshot 内容，且结尾不再有 Assistant 预占消息。脚本必须显式调用 `process.exit(0)`。

---

### 🤖 Coding Agent C: 逻辑引擎升级专家 (Phase 3)
**项目背景**：实现 SLE 决策结果的结构化解析，提高工具调用与直答的稳定性。
**核心参考**：`src/agent/sle.ts`。
**任务详述**：
1.  **修改 `run` 方法**：
    -   在调用 `openai.chat.completions.create` 的 `payload` 中，增加 `response_format: { type: 'json_object' }`。
    -   禁用 `stream: true` (逻辑判定阶段优先保证 JSON 完整性，由 SLC 负责首块垫词秒回)。
    -   **解析响应**：等待完成并 `JSON.parse`。
    -   **字段分发**：将 `parsed.thought` 输出到日志；将 `parsed.response` 赋值给 `sleFullOutput`。
    -   **兼容 Tools**：保持对 `response.choices[0].message.tool_calls` 的原样处理，确保功能不退化。
2.  **强制闭环**：编写 `verify_v3.6.2_p3.ts`，模拟 LLM 回复一个 JSON 对象，验证 `sleFullOutput` 是否正确抓取了 `response` 字段。脚本必须显式调用 `process.exit(0)`。

---

### 🤖 Coding Agent D: 回归验证专家 (Phase 4)
**项目背景**：V3.6.2 核心校准完成后的整体质量闭环。
**核心参考**：`overview.md`, `doc/design/3.6/v3.6.2_payload_role_rectification.md`。
**任务详述**：
1.  **全链路回归**：通过真实的任务意图 (如：查询天气)，观察 `.llm_requests.log` 中的 Scenario B Payload 布局是否符合分层标准。
2.  **更新文档**：更新 `overview.md` 中的架构图或文字描述，明确 `DECIDING` 场景的消息分层规范。
3.  **结果审计**：确保在工具判定期间，SLE 的 `content` 始终为 JSON 格式，且不再受到语音层垫词干扰。
**强制闭环**：输出 `v3.6.2_final_report.md` 总结本次重构的架构收益，并执行验证确保脚本全绿通过。
