# Fast Agent V3.5：声明式动态 Skill 架构 - 开发、测试与迁移计划

这份文档详述了如何把之前的硬编码试验产物平滑过渡，并将系统彻底升级到**“基于文件夹结构热加载的动态声明式 Skill (Declarative Folder-based Skills)”**体系。

---

## 第一阶段：伪业务清理与双轨制注册中心改造 (Phase 1)
**目标：保留不可动摇的原生底座，扬弃所有硬编码长效业务，实现混合体加载引擎。**

### 1. 废弃硬编码包袱
删除之前在试验期的、写死在 `src/agent/skills/core/` 目录里的 `sleep-skill.ts` 或 RAG 模拟相关的 `.ts` 业务文件。仅保留原生的骨架与灵魂：
- `asr-correction.ts` (热词防御原生骨架)
- `task-delegate.ts` (意图委派原生骨架)

### 2. 双轨制注册架构修改
改造 `src/agent/skills/index.ts` 中的 `SkillRegistry`：
1. 建立 `registerCoreSkills()` 方法，显式装载第一步保留的原生核心。
2. 建立 `loadFromDirectory(dirPath)` 框架钩子，预留为后续遍历 `skills_repo` 服务。这是将静态 TS 与动态 YAML 统一抹平存放进同一个 `Map<string, IFastSkill>` 中的枢纽层。

### 🚨 测试与验证（必须满足）
执行现有的自动化测试工具：`npx tsx verify_asr_v3.4.ts`
**期望结果**：测试绿灯。原生能力丝毫未受影响。

---

## 第二阶段：声明式解析引擎落地 (Phase 2)
**目标：在不破坏现有流转逻辑下，实现基于 YAML 的外部 Skill 扫描与挂载。**

### 1. 外挂层转译核心
1. **DynamicSkillWrapper**：在 `src/agent/skills/` 下创建一个实现 `IFastSkill` 契约的标准外壳。它通过接收读取到的 YAML 对象初始化自己。
2. 完善 `SkillRegistry.loadFromDirectory()`：引入 node `fs` 和 `js-yaml`，真实地遍历根目录下新建的 `skills_repo/` 文件夹。把合法的子文件夹下的 `skill.yaml` 全部解析转为 `DynamicSkillWrapper` 然后 `register()` 它。

### 🚨 测试与验证
配置一个合法的 `skills_repo/test_skill/skill.yaml`。
执行：`npx tsx verify_skill_loader.ts`，验证 `SkillRegistry` 是否成功捕捉并转换了假文件夹内的配置并在大模型的 `tools` Schema 数组里输出了这柄伪造武器。主系统心跳绝不能死锁。

---

## 第三阶段：异步流转与真实动作转译执行 (Phase 3)
**目标：实现真异步非阻塞抽离，并在 Wrapper 内将参数推向外部 MCP Server。**

### 1. 异步游离心脏改造
在 `ToolResultHandler.handleToolCalls` 中，只要遇见 `isLongRunning: true` (无论是原生还是外挂 Wrapper)，直接通过 `Promise.resolve().then()` 塞入异步队列，彻底解绑主心跳阻塞，并在耗时执行结束后由 `appendCanvasAudit` 进行无痕写回恢复。

### 2. Wrapper 请求对接
完善 `DynamicSkillWrapper.execute()` 业务逻辑，依据 yaml 给定的地址，组装 axios/fetch 连带模型返回的参数一并发送至指定的 URL 请求结果（如外部标准的 MCP Service）。

### 🚨 测试与验证
执行 `npx tsx verify_async_mcp.ts`。配置一个 3 秒延迟的模拟天气接口配置。要求大喊之后 TTFT < 800ms 秒回极速安抚，然后在 3 秒后控制台准时打印背景画布收到天气知识的成功事件流。

你当前在维护 OpenClaw RTC Fast Agent (V3.5)。我们现在决定彻底拥抱类似 Claude Code 的“声明式文件夹外挂技能 (Declarative Folder-based Skills)”模式，需要把你以前为了验证思路手涂的业务 `.ts` 彻底删除，重塑纯粹底座。原则是：**核心隔离，极简至上**。

## 2. 需参考核心文档/代码
- 过渡计划: `doc/design/3.5/v3.5-skill-dev-plan.md`
- 代码目标: `src/agent/skills/index.ts` (Registry), `src/agent/tool-result-handler.ts` (调用侧)，以及整理 `skills/core/` 目录。

## 3. 开发详细计划
**不要添加任何新的业务功能代码！**
1. 删代码：删除 `src/agent/skills/core/` 里面我们后来捏造的所有和业务相关的 TS 文件（比如 `sleep-skill.ts`, `rag-document.ts` 等），只保留 `asr-correction.ts` 与 `task-delegate.ts` 两个核心底座。

   - 新增：`registerCoreSkills(executor, summarizer, callManager)`，在这里把原生核心装进字典。
   - 新增：打出 `loadFromDirectory(dirPath)` 这支桩（只需定义为空方法并写好注释）。
3. 调整初始化入口：将 `tool-result-handler.ts` 里的注册代码重构一下去调用 `SkillRegistry.getInstance().registerCoreSkills(...)`，避免硬编码乱挂载。

## 4. 单元测试要求
你需要终端执行现有的高压验收脚本：`npx tsx verify_asr_v3.4.ts`。
观察执行脚本是否有错。你的重构清理绝对不能破坏原有的任何系统意图与防御链。必须纯绿灯 (Zero Regression)！

### Phase 2: 声明式解析器基建装配
```markdown
# 任务目标: V3.5 第二阶段 - 声明式 Skill 配置树遍历加载器

## 1. 项目背景
继上一阶段骨架双轨制搭好后，我们要实现对挂载盘的彻底热力扫描。系统启动时需主动扫描特定挂载目录 `skills_repo/`，将里面的 YAML 读出，利用转译外壳挂进内存供 SLE 调用。对标最新模型上下文扩展技术。

## 2. 需参考核心文档/代码
- 架构设计: `doc/design/3.5/3.5-Skill支持.md`
- 代码目标: `src/agent/skills/index.ts` 与新建立的外壳 `src/agent/skills/DynamicSkillWrapper.ts`。

## 3. 开发详细计划
1. 设计武器舱外包装：新建 `src/agent/skills/DynamicSkillWrapper.ts`，基于 `IFastSkill` 实现。构造函数中需要接收一段提取出的 YAML 参数配置（包括 name, description, parameters, isLongRunning 等）。`execute` 先 mock 占位。
2. 文件探测逻辑：在 `SkillRegistry` 充实前文留下的 `loadFromDirectory(path)`：引入 Node 的 `fs.promises` 和 `js-yaml`，扫描遍历子文件夹里的 `skill.yaml`。组装为 `DynamicSkillWrapper` 注入进 Registry 的武器库 Map。

## 4. 单元测试要求
你需要自行编写 `openclaw-test-env/verify_skill_loader.ts`。
在根目录伪造一个合规的 `skills_repo/test_skill/SKILL.md` 测试组件。使用该脚本初始化 Registry 并扫盘，要求必须打印出抓取信息。
```

### Phase 3: 异步脱轨与 MCP 动作转译执行
```markdown
# 任务目标: V3.5 第三阶段 - 异步微任务解堵塞与真实 MCP 请求派发

## 1. 项目背景
最后冲刺。声明外壳有了，现在要让 `DynamicSkillWrapper` 真正发挥作用。不仅要在收到指令时发真实网络请求调外部接口（MCP），还绝对不能因为 HTTP 等待导致系统极速秒回的优势丧失（TTFT < 800ms）。

## 2. 需参考核心文档/代码
- 设计要求: `doc/design/3.5/3.5-Skill支持.md`
- 代码目标: `src/agent/tool-result-handler.ts` (核心拦截) & `DynamicSkillWrapper.ts` (执行器)。

## 3. 开发详细计划
1. 解死锁重构：在 `ToolResultHandler.handleToolCalls` 遍历动作时，抓取到拥有 `isLongRunning: true` 的技能。强制绕开等待，采用 `Promise.resolve().then(...)` 挂为宏/微任务抽离。主心跳方法需要瞬间通过不阻塞。
2. 背景治愈上链：异步等待请求返回结果后，用 Summarizer 精简内容，显式利用 `canvasManager.appendCanvasAudit()` 挂载结果让守护进程通知用户。
3. 真实 Endpoint 打交道：完善 `DynamicSkillWrapper.execute`：根据 YAML 提取的 URL，向远程服务器发真实请求并拉回。

## 4. 单元测试要求
1. 需要模拟拉起一个 3 秒延迟的假的外部服务（测试环境造一个 http server）。
2. 在 `skills_repo/weather_mcp/` 放下带有具体 url 指向改服务配置的 `SKILL.md`。
3. 新建 `openclaw-test-env/verify_async_mcp.ts`。执行后断言：心跳打印全覆盖毫无阻塞，并在 3 秒大限来临时准确无误看到 Audit Log “背景挂载成功”的长文。
```
