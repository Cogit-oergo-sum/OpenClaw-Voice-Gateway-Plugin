# Fast Agent V3.1 实施计划

## 1. 概述
V3.1 版本专注于完成 V3 设计文档中定义的 **提示词拼装服务 (Prompt Assembly Service)** 和 **分层提示词架构 (Layered Prompting Architecture)**。主要目标是解决当前 SLC 实现中存在的“语义缝合断裂”和“人设一致性不足”的问题。

## 2. 核心特性 (V3.1)

### 2.1 分层提示词架构 (Layered Prompting)
- **核心内核层 (Core Soul Layer)**：定义在 `soul.md` 和 `IDENTITY.md` 中。这是不可撼动的人设滤镜，定义 Jarvis 的语气、称谓（先生）及价值观。
- **技能插件层 (Skill Plugin Layer)**：定义在 `AGENTS.md` 及其他技能文档中。包含逻辑、规则和事实数据。
- **动态拼装**：升级 `ShadowManager` 为“提示词拼装器”，根据调用者是 SLC（低延迟、强人设）还是 SLE（高智力、强规则）来选择性合并层级。

### 2.2 SLC 动态语义缝合 (Stitching Protocol)
- **已播报感知 (Delivered Text Awareness)**：SLC 将真正使用 `deliveredText`（即并联双路赛跑中前台先说的那半句话）来衔接后续回复。
- **潜意识注入 (Shadow Injection)**：
  - 针对 `__INTERNAL_TRIGGER__`（由 Watchdog 触发的播报），将注入临时的引导块：
    `(潜意识思考: 刚才我已替主脑开了个头说了: "${deliveredText}"。现在画布显示结果是: "${canvasSummary}"。我必须保持 Jarvis 的优雅口吻，从锚点自然接续，严禁道歉或复读。)...`
  - 该机制在不污染长期记忆的前提下，告知 LLM 当前的交互锚点。

### 2.3 持久化脱敏 (Persistence & Filtration)
- **物理审计**：所有包含 `(潜意识思考: ...)` 的完整对话将记录在 `.trace.jsonl` 中供复盘。
- **上下文过滤**：在拼装下一轮对话历史时，必须执行静态过滤，剥离所有潜意识块，确保传递给 LLM 的是干净、优雅的 Jarvis 正式对白。

### 2.4 Canvas 版本与同步协议
- 在 Watchdog 中实现 `version`（时间戳）比对机制。
- 确保 `runSLE` 在工具执行期间严格标记 `status: PENDING`，在总结完成后才恢复 `READY`。
