# Implementation Plan - Agent 3A: SLC Shadow Thought Upgrade & Watchdog Aggregation

本文档描述了 SLC（交互魂魄引擎）潜意识注入逻辑的升级方案，以支持多任务状态的聚合播报及状态感知。

## 1. 核心改动点

### 1.1 `src/agent/prompts.ts`
- **签名改造**: `buildShadowThought(type: ShadowThoughtType, tasks: TaskItem[])`。
- **逻辑分支**:
    - `RESULT_DELIVERY`: 聚合已完成/失败任务的摘要（`direct_response` 或 `summary`）。
    - `PROGRESS_REPORT`: 聚合 PENDING 任务的进度（`progress_detail` 或 `stage`）。
    - `chat`/其他: 聚合任务的基础事实信息。
    - **降级处理**: `tasks` 为空时返回通用提示。

### 1.2 `src/agent/slc.ts`
- **接口修改**: `run()` 方法参数 `directResponse: string` -> `tasks: TaskItem[]`。
- **注入逻辑**: 调用新版 `buildShadowThought` 并传入任务数组。
- **类型识别**: 根据任务状态（是否全部完成）自动切换 `RESULT_DELIVERY` 或 `PROGRESS_REPORT`。

### 1.3 `src/agent/agent-orchestrator.ts`
- **适配调用**: 在 `orchestrate()` 中，收集目标 `TaskItem` 数组并传递给 `slc.run()`。
- **情境过滤**:
    - `__INTERNAL_TRIGGER__`: 传递目标任务数组。
    - `__TOOL_WAITING_TRIGGER__`: 传递待处理任务。
    - 正常问答: 传递 `matched_task_ids` 对应的任务。

### 1.4 `src/agent/watchdog.ts` & `src/agent/fast-agent-v3.ts`
- **Watchdog**: 确保 `trigger` 事件携带完整的 `TaskItem[]`。
- **FastAgentV3**: `handleWatchdogTrigger` 适配新参数，并确保播报成功后调用 `markAsDelivered`。

## 3. 风险评估
- **Token 消耗**: 聚合多个任务的摘要可能导致 SLC 的 Pre-fill 变长。
- **播报延迟**: 聚合逻辑若过于复杂，可能轻微增加 TTFT。

## 4. 验证方案
开发者需执行 `npx ts-node verify_sript/verify_v3.7_3a_slc_shadow.ts` 以确保：
1. 潜意识生成内容符合预期。
2. Watchdog 聚合播报链路畅通。
3. 播报后任务状态正确标记。
