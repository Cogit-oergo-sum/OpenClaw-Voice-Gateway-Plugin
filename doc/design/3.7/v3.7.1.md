# OpenClaw V3.7.1: Persona, Broadcast & Routing Refinement

## 1. 整体背景与目标 (Background)
在 V3.7 多任务架构大版本上线后，由于 SLC 与 SLE 的过度解耦，导致了人设回归、播报复读以及复杂总结意图识别失效的问题。V3.7.1 旨在通过异步人设同步、投递原子化及启发式路由策略，补全多任务体验的最后一块拼图。

## 2. 核心架构变更 (Architectural Changes)
### 2.1 Persona 召回 (REFINING Scenario)
- **机制**：在 `AgentOrchestrator` 中引入异步 `REFINING` 场景。
- **触发频率**：仅在 Session 启动 (`isNewSession`) 或 `compact_persona` 尚未生成时触发一次后台提炼。
- **闭环**：提炼出的高密度人设将持久化至 `ShadowState` 的 `metadata.compact_persona`，并作为 SLC 后续所有回复的 System Prompt 基座。

### 2.2 播报原子化 (Broadcast Atomicity)
- **改进点**：在 `WatchdogService` 触发 `trigger` 事件前，强制将匹配任务标记为 `is_delivered: true` 并执行 `persistContext`。
- **防护点**：引入 `updated_at` 时间戳，防止由于磁盘快照回滚导致的陈旧 READY 状态产生复读。

### 2.3 启发式意图路由 (Heuristic Routing)
- **总结重定向**：将模糊的总结请求强制映射至 `delegate_openclaw` 工具进行长文处理。
- **指代消解**：在 `ROUTING` 阶段优先向 LLM 推荐最近更新的任务结果，减少 CLARIFY 的触发频率。
