# Fast Agent 核心实现分析报告 (V2.3.0)

本文档记录了当前 Fast Agent 的底层架构设计与代码实现细节，旨在为后续维护与功能演进提供参考。

---

## 1. 核心架构：双魂并联 (Dual-Soul Parallel)
Fast Agent 的核心是一套名为 **“影子锚点 (Anchor Reflection) 语意缝合”** 的双魂架构，重点解决语音交互中大模型响应延迟与自然度之间的冲突。

### 1.1 SLC (Small Language Component - 本能魂)
*   **定位**：毫秒级响应节点。
*   **职责**：在用户话音刚落的 **300ms 内** 给出非实质性的“开口语”（如：“先生，”、“噢？”、“这个嘛...”）。
*   **技术目标**：抢占音频链路，通过极速反馈消除用户的“死寂感”。

### 1.2 SLE (Small Language Engine - 逻辑魂)
*   **定位**：逻辑中枢与工具调度。
*   **职责**：执行深度语境分析、工具调用（如 `delegate_openclaw`）及长文本内容生成。
*   **缝合机制**：SLE 在启动时会将 SLC 已播报的内容作为 `assistant` 角色的 `prefill` 内容传入。模型从该锚点开始无缝续写，确保语意连贯且不重复。

## 2. 核心组件与机制

### 2.1 影子管理器 (Shadow Manager)
*   **持久化**：管理工作区中的通话状态与对话历史。
*   **语境对齐**：在每轮对话开始前，自动加载最近的对话背景与 **"Jarvis"** 人格定义。
*   **潜意识管理**：维护 `Shadow` 状态，确保 SLE 能够感知当前环境的变化。

### 2.2 动态哨兵控制 (Watchdog System)
针对网络波动或主脑执行耗时工具时的真空期，系统内置了 Watchdog 机制：
*   **时间阈值**：普通对话 2.5s 阈值，工具调用 5s 阈值。
*   **动作**：若 SLE 未能准时吐字，哨兵自动注入垫词（Filler），维持对话活性。
*   **话术动态选择**：
    *   *常规场景*：“让我想想...”、“先生，我在理解...”
    *   *工具场景*：“主脑反馈稍慢，我盯着呢...”、“请稍等，我正在同步细节...”

### 2.3 背景任务与委派机制
通过 `delegate_openclaw` 实现与主 Agent 的协同：
*   **实时处理**：任务在 5s 内完成，直接播报结果。
*   **后台模式 (Background Mode)**：任务超过 5s，Agent 会自动通过转场话术（“这件事有点复杂，我交托主脑去办了，您可以先忙别的”）告知用户。
*   **异步通知**：任务最终完成后，通过 `notifier` 回调触发 TTS 播报或发送字幕提醒。

## 3. 性能优化手段

| 优化项 | 实现方式 | 预期收益 |
| :--- | :--- | :--- |
| **连接预热** | 每 50s 发送一个 `.` 探测包保持长连接 | 减少 TCP/TLS 握手及 TTFT 延迟 |
| **流式塌缩** | 在 `chat-api.ts` 中移除换行符与 Markdown 符号 | 确保字幕在移动端不被切割，音画同步 |
| **异步 I/O 分离** | 影子恢复与 SLC 响应并行，不互相阻塞 | 极致缩短首字响应时间 (First Token) |
