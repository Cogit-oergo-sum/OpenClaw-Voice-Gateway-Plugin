# 02. 个性化配置与高阶开发开发指南 (Customization)

本章节面向对 OpenClaw RTC 插件有**深度定制需求**的开发者。你将学会如何自由魔改提示词，并彻底掌握**通过极简配置文件，给 AI 热拔插海量第三方技能（MCP）**的超强架构能力。

---

## 🎭 1. AI 提示词与潜意识（Prompt）定制

Fast Agent V3 系的架构坚决贯彻**“指令与系统逻辑彻底分离”**。这意味着你要赋予模型怎么样的底层人设、有什么样不可触碰的底线，完全不需要修改代码：

👉 **文件位置：** `src/agent/prompts.ts`

1. **`SYSTEM_FAST_PROMPT`**：每次对话模型都会看到的第一句话（全局人格基座）。
2. **`SLE_ACTION_PROTOCOL`**：后台逻辑代理在进行行为调度和思考分流时的决策树说明。如果你想增加特殊的指令流转，请修改此处。
3. **影子思维控制（Shadow Thoughts）**：决定底层系统的诸如 `[任务完成]` 等提示在最终语音合成时采用何种态度过渡。

---

## 🛠️ 2. 热装配外挂技能！(Declarative Skills Folder)

**V3.5 系统的最强绝杀！** 你不需要编写一堆冗余的 TS 类来扩展外网服务接入。我们彻底采用了与行业前沿（如 Claude Code）对齐的**“声明式结构”**。

如果你想在这个网关上追加一个【查询天气】MCP 服务，或者追加【查询公司本地数据库】的功能，操作如下：

### 第一步：在 `skills_repo` 创建一个特征文件夹
打开位于根目录的扩展卡槽文件夹 `skills_repo/`，新建属于你技能的名字，例如 `weather_query/`。

### 第二步：丢进去你的 `SKILL.md` 配置说明书
系统会自动解析这个文件夹。我们采用的是与 Claude Code 完全一致的 **「YAML Frontmatter + Markdown」** 双体合一格式：
你在头部书写结构化的元数据与请求接口，在主体用大白话书写你的要求：

```markdown
---
# ✅ 给大模型看的标识名称，不要有空格
name: query_local_weather

# ✅ 告诉底层逻辑大模型，什么语境下抓取用户意图并调度这个技能
description: 当用户提及天气、出门穿搭时，查询目的地的气候数据。

# ✅ 是否为耗时任务。设置为 true 后系统自动脱轨异步化，坚决不阻塞用户语音的交互速度！
isLongRunning: true

# ✅ 设置具体这个动作被触发时要请求的外端点（MCP或本地执行器）
execution:
  type: http
  url: "http://127.0.0.1:8080/mcp/weather"
  method: POST

# ✅ 告诉大模型你需要它回传给你接口什么样的入参 (JSON Schema)
parameters:
  type: object
  properties:
    city:
      type: string
      description: 中国某城市名（限拼音，如 "shanghai"）
  required:
    - city
---

# 气象摘要指南 (Summarizer Prompt)
当以上 HTTP 接口成功返回大量冷冰冰的气象 JSON 后，请你扮演一个贴心播报员。
你不要一口气念完所有气温，挑选最核心的高低温，并根据结果返回一句类似于：“天气数据已查明。北京今天比较冷哦，需要我为您展开说光明天的穿衣建议吗？”的过渡话术。
```

就这么简单！我们把呆板的 `yaml` 和 `prompt` 完美合二为一，所见即所得。

### 第三步：享受成果，无需重启 🌟

当下一次请求进入系统，调度扫描器就已经把你的这个文件夹里的声明转化为了大模型眼中的 **Tool Call**，并全自动与底层 HTTP MCP 网络对接完毕了。这对于你实现各类物联网设备桥接或是知识库桥接，效率都趋近于无敌。
