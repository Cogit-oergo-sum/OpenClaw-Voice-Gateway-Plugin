# Voice Gateway Plugin 需求文档

> OpenClaw 语音通话插件 — 让 OpenClaw 具备通过 ZEGO RTC 进行实时语音通话的能力

---

## 1. 项目背景

OpenClaw 是一个开源的自主 AI Agent 运行时，但目前仅支持文本交互（Chat Channel）。用户希望通过一个 Web 网页，直接与自己的 OpenClaw 进行语音电话，实现更自然的实时语音交互。

**核心依赖产品：**

| 产品 | 角色 | 说明 |
|------|------|------|
| **OpenClaw** | Agent 运行时 | 部署在云端，承载用户的 AI Agent，包含人格（SOUL）、记忆（MEMORY）、用户画像（USER）|
| **ZEGO AI Agent** | 语音链路 | 提供 ASR（语音转文本）、TTS（文本转语音）能力，支持自定义 LLM 代理 |
| **ZEGO RTC** | 实时通信 | Web 端音频采集/推流/拉流，ZEGO Express SDK |

---

## 2. 目标用户

已部署 OpenClaw 到云服务器的用户。期望以最低接入成本（安装 Plugin + 配置密钥）获得语音通话能力。

---

## 3. 核心需求

### 3.1 快速接入

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-01 | 以 OpenClaw Plugin 形式提供，`npm install` 后即可使用 | P0 |
| R-02 | 用户仅需在 Plugin 配置中填写 ZEGO AppID、AppSign、ServerSecret、AI Agent 密钥和 LLM API Key | P0 |
| R-03 | 提供配套的 Web 前端页面，集成 ZEGO Express SDK，支持发起/挂断语音通话 | P0 |

### 3.2 语音通话基础能力

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-04 | 用户通过 Web 页面发起语音通话，与 OpenClaw 的 AI Agent 进行实时语音对话 | P0 |
| R-05 | AI 回复延迟指标（从用户说完到听到 AI 首个音节）：纯文本 P95 ≤ 1.5s，含工具调用 P95 ≤ 2.0s，降级兜底 P99 ≤ 1.5s。其中 Plugin 侧（含 LLM 推理）控制在 1 秒内 | P0 |
| R-06 | AI 回复支持流式语音输出（边生成文本边合成语音），减少感知延迟 | P0 |
| R-07 | 支持自然语音打断（用户说话时 AI 停止当前输出，ZEGO 内置能力） | P1 |

### 3.3 通话生命周期管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-08 | Plugin 暴露通话管理 API（发起通话、挂断通话、查询状态），Web 前端通过这些 API 控制通话 | P0 |
| R-09 | Plugin 内部维护通话状态（AgentInstanceId、RoomId、通话开始时间），支持多用户隔离 | P0 |
| R-10 | 通话结束时自动调用 ZEGO `DeleteAgentInstance` 清理资源 | P0 |
| R-11 | 对话日志采用内存缓冲 + 通话结束统一归档策略：通话中对话存内存，结束时一次性写入 `memory/` 日志。每 5 分钟定时持久化作为断线保护 | P0 |
| R-12 | 异常断线处理（Web 端监听 RTC 断连事件，支持重新发起通话） | P1 |
| R-13 | 利用 LLM Prompt Caching（API 参数显式声明）降低延迟和成本。System Prompt 拆分为稳定前缀和动态后缀 | P1 |

### 3.4 人格与记忆一致性

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-14 | 语音通话中的 AI 回复必须保持与 OpenClaw `SOUL.md` 中定义的人格一致 | P0 |
| R-15 | 语音通话中的 AI 能感知 `USER.md` 中的用户画像信息 | P0 |
| R-16 | 语音通话中的 AI 能感知 `MEMORY.md` 和近期 `memory/` 日志中的历史上下文 | P0 |
| R-17 | 语音通话的对话记录自动写入 `memory/` 日志，供 OpenClaw 主 Agent 后续读取 | P0 |
| R-18 | SOUL.md 内容精简策略：截取前 N 行（由配置项 `soulMaxTokens` 控制），建议用户将核心人格特质写在 SOUL.md 开头 | P0 |

### 3.5 快速响应 Agent

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-19 | Plugin 内嵌快速响应 Agent（直连快速 LLM），负责实时语音对话的所有响应 | P0 |
| R-20 | **ASR 纠错 - 第 1 层：LLM 上下文推断纠错。** System Prompt 指令让 LLM 自动修正同音字错误（如"会一"→"会议"），无需用户感知，零额外延迟 | P0 |
| R-21 | **ASR 纠错 - 第 2 层：动态热词学习。** 当 LLM 无法仅凭上下文纠错时，通过对话澄清 + 搜索/RAG 确认正确词汇后，运行时调用 ZEGO `UpdateAgentInstance` API 动态追加 ASR 热词，并在会话上下文中建立别名映射，后续 ASR 和 LLM 双重保障识别正确 | P1 |
| R-22 | 支持配置 ZEGO ASR 静态热词（`ASR.Params.hotword_list`），作为基础热词在通话创建时加载 | P0 |
| R-23 | 快速响应 Agent 支持调用内置工具（天气查询等）完成简单任务 | P1 |
| R-24 | 快速响应 Agent 能判断任务复杂度，将复杂任务异步委托给 OpenClaw 主 Agent | P0 |
| R-25 | 快速响应 Agent 超时时（>900ms 无输出），自动返回兜底回复 | P0 |
| R-26 | 超时降级区分：零输出时发送完整兜底回复；已有部分输出时不追加额外兜底文本 | P0 |
| R-27 | 多轮对话上下文裁剪：当 ZEGO 传入的 messages 历史 + Plugin 注入的上下文总量超过阈值时，仅保留 system message + 最近 N 轮对话 | P0 |

### 3.6 OpenClaw 主 Agent 联动

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-28 | 复杂任务通过 Webhook 异步委托给 OpenClaw 主 Agent，不阻塞语音通话。委托消息使用结构化模板格式 | P0 |
| R-29 | OpenClaw 主 Agent 的任务结果写入 Memory 日志，快速 Agent 在后续对话中自动感知。需通过 SKILL.md 指示主 Agent 完成任务后必须将结果摘要写入 memory 日志 | P0 |
| R-30 | 为 OpenClaw 主 Agent 注册 `voice_speak` 工具，使主 Agent 能主动触发语音播报 | P1 |
| R-31 | 为 OpenClaw 主 Agent 注册 `voice_status` 工具，使主 Agent 能查询通话状态 | P2 |

### 3.7 主动通知与播报

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-32 | Plugin 后台服务监控通知事件，在通话进行中主动播报（如日程提醒） | P1 |
| R-33 | 主动播报支持优先级控制（High/Medium/Low）和相同优先级打断策略（ClearAndInterrupt/Enqueue） | P2 |

## 4. 用户场景

### 场景 0：通话生命周期

> 用户从发起通话到挂断的完整流程。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户打开 Web 页面，点击"发起通话" | Web 调用 Plugin API `POST /voice/start-call` |
| 2 | Plugin 调用 ZEGO API 注册智能体 + 创建实例 | 返回 RoomId、Token、AgentInstanceId |
| 3 | Web 端通过 ZEGO Express SDK 加入 RTC 房间 | 通话建立，可开始说话 |
| 4 | 用户进行多轮语音对话（场景 1-5） | 正常交互 |
| 5 | 用户点击"挂断" | Web 调用 Plugin API `POST /voice/end-call` |
| 6 | Plugin 调用 ZEGO `DeleteAgentInstance` | 资源清理 |
| 7 | Plugin 将本次通话对话记录归档写入 memory 日志 | 主 Agent 后续可感知 |

**异常场景：**
- 网络断线 → Web 监听 RTC 断连事件 → 提示用户重新发起
- ZEGO 实例异常 → Plugin 收到 ZEGO 服务端异常回调 → 清理状态 + 通知 Web 端

**关键验证点：** 通话建立到可说话 < 3s，挂断后资源完全清理。

---

### 场景 1：纯闲聊快速回复

> 用户在语音通话中随意聊天。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说"今天心情不错" | ASR 转文本 → Plugin 收到请求 |
| 2 | 快速 Agent 判断为闲聊 | 无需工具调用，直接生成回复 |
| 3 | doubao-1.8-flash 流式输出 | ~500ms 用户听到 AI 回复 |
| 4 | 对话记录写入 memory 日志 | OpenClaw 主 Agent 可感知 |

**关键验证点：** 回复保持 SOUL.md 人格、回复简洁（1-3 句）、P95 < 1.5s。

---

### 场景 2：调用内置工具（天气查询）

> 用户问天气情况。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说"明天北京天气怎么样" | ASR 转文本 |
| 2 | 快速 Agent 判断需要调用 `get_weather` 工具 | doubao 返回 tool_call |
| 3 | Plugin 调用天气 API（~100ms） | 获取结构化天气数据 |
| 4 | 将工具结果作为上下文，第 2 次调用 doubao | 生成自然语言回复 |
| 5 | 流式输出 "明天北京晴转多云，15度..." | ~800ms 用户听到回复 |

**关键验证点：** 工具调用 + 第 2 次 LLM 总耗时 < 1s，P95 < 2s。

---

### 场景 3a：ASR 纠错 — LLM 上下文推断

> 用户语音中包含常见同音字错误，LLM 可基于上下文直接推断正确含义。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说"明天会议几点"，ASR 识别为"明天会一几点" | Plugin 收到带错字的文本 |
| 2 | 快速 Agent 在 System Prompt 指令下自动纠错 | 上下文中"几点"暗示时间相关，推断"会一"→"会议" |
| 3 | 正常回复会议时间 | 用户无感知纠错发生 |

**关键验证点：** 纠错无额外延迟（Prompt 工程实现），不告知用户做了纠错。

---

### 场景 3b：ASR 纠错 — 动态热词学习

> ASR 识别了一个 LLM 也无法仅凭上下文修正的专有名词，用户通过后续对话提供线索让 AI 学会正确词汇。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说"你知道周杰伦吗" | ASR 识别为"周洁纶" |
| 2 | 快速 Agent 不确定"周洁纶"是谁 | 回复"你说的周洁纶是？" |
| 3 | 用户说"是一个歌手" | ASR 正常识别 |
| 4 | Agent 结合"歌手"线索 + 调用搜索工具 | 搜索"周洁纶 歌手"→ 命中"周杰伦" |
| 5 | Agent 确认正确词汇 | 回复"哦你说的是周杰伦！我知道..." |
| 6 | Plugin 调用 `update_asr_hotwords` 工具 | ZEGO `UpdateAgentInstance` 追加热词 "周杰伦\|10" |
| 7 | Plugin 在会话上下文中记录别名映射 | `{"周洁纶": "周杰伦"}` |
| 8 | 后续用户再说"周杰伦" | ASR 热词生效，正确识别 |
| 9 | 即使 ASR 仍识别为"周洁纶" | LLM 通过别名映射也能正确理解 |

**关键验证点：**
- 热词更新后，后续 ASR 识别准确率提升
- 别名映射作为双重保障：即使 ASR 仍错，LLM 也能纠正
- 热词仅在本次通话生效（随 AgentInstance 销毁而清除）

---

### 场景 4：异步委托 OpenClaw 主 Agent

> 用户请求复杂任务，如安排日程、发邮件。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说"帮我给张三发一封会议邀请邮件" | ASR 转文本 |
| 2 | 快速 Agent 判断为复杂任务 | 调用 `delegate_openclaw` 工具 |
| 3 | POST `/hooks/agent` 异步通知 OpenClaw（~20ms） | 消息使用结构化模板格式 |
| 4 | 快速 Agent 立即回复"好的，我来帮你处理" | ~400ms 用户听到回复 |
| 5 | OpenClaw 主 Agent 在后台执行邮件发送任务 | SKILL.md 指示完成后写 memory |
| 6 | 用户稍后问"邮件发了吗" | 快速 Agent 读取 memory，获知任务状态 |

**关键验证点：** 异步委托不阻塞语音，后续对话能感知任务结果。

---

### 场景 5：超时降级

> LLM 或工具调用异常导致超时。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户说了一个复杂问题 | ASR 转文本 |
| 2 | doubao 推理时间超过 900ms 且无 SSE 输出 | Plugin 触发零输出降级 |
| 3 | Plugin 立即返回兜底回复"嗯，让我想想..." | 用户不会感觉"死线" |
| 4 | 后台继续处理，结果写入 memory | 下一轮对话时带上结果 |

**200ms 已有部分输出的场景：**
- doubao 已输出 "明天北" 然后卡住 → 不追加兜底文本
- ZEGO TTS 处理已收到的文本片段 → 用户听到部分内容

**关键验证点：** 零输出时 P99 < 1.5s 听到兜底；部分输出时不产生语义混乱。

---

### 场景 6：OpenClaw 主 Agent 主动语音播报

> OpenClaw 的定时任务或其他 Channel 的消息需要通知用户。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | OpenClaw 主 Agent 调用 `voice_speak` 工具 | 或后台服务检测到通知文件 |
| 2 | Plugin 获取当前通话的 AgentInstanceId | 从通话状态管理器获取 |
| 3 | 调用 ZEGO `SendAgentInstanceTTS` API | 主动播报"提醒您，14点有会议" |
| 4 | 用户在通话中直接听到通知 | 不打断正常对话流 |

**前置条件：** 当前有活跃通话。无活跃通话时 → 通知暂存或忽略。

**关键验证点：** 通知不影响正在进行的对话，支持优先级控制。

---

### 场景 7：上下文跨通道共享

> 用户在文本 Channel 和语音通话之间切换，期望 AI 保持上下文连贯。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户通过 WhatsApp 告诉 OpenClaw "我明天要去出差" | OpenClaw 主 Agent 记录到 memory |
| 2 | 用户发起语音通话，说"帮我查一下明天出差目的地的天气" | 快速 Agent 读取 memory |
| 3 | 快速 Agent 从 memory 获知出差目的地 | 调用天气工具查询 |
| 4 | 回复目的地天气信息 | 上下文连贯 |

**关键验证点：** 语音通话能读取其他 Channel 的交互记录。

---

### 场景 8：多轮对话上下文膨胀

> 用户在一次通话中聊了很多轮。

| 步骤 | 行为 | 预期 |
|------|------|------|
| 1 | 用户已经进行了 15 轮对话 | ZEGO 传入 messages 包含历史 |
| 2 | Plugin 收到 messages 包含 system + 15 轮历史 | 上下文总量较大 |
| 3 | Plugin 的上下文裁剪策略生效 | 仅保留注入的 system + 最近 10 轮 |
| 4 | 回复速度不受多轮对话影响 | 延迟仍在 1s 内 |

**关键验证点：** 20 轮对话后延迟不显著增长。

## 5. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NR-01 | 延迟 | Plugin 侧处理延迟 ≤ 1000ms（P95） |
| NR-02 | 安全 | HTTP Handler 需 Token 认证 + HTTPS，ZEGO ServerSecret 不暴露到前端 |
| NR-03 | 可配置 | ZEGO 密钥、LLM 配置、TTS 音色、ASR 厂商/热词等通过 Plugin 配置文件管理 |
| NR-04 | 可观测 | 记录每轮对话的延迟、工具调用、降级事件等日志。通话结束时记录 ZEGO 返回的延迟统计数据 |
| NR-05 | 兼容性 | 兼容 OpenClaw 最新版本的 Plugin SDK |
| NR-06 | 并发 | 默认 ZEGO 账号最多 10 个并发实例，文档中注明此限制 |

---

## 6. 不在范围内（Out of Scope）

- ❌ 视频通话 / 数字人形象
- ❌ 多人与同一 Agent 同时语音通话
- ❌ 移动端原生 App（仅 Web）
- ❌ OpenClaw 主 Agent 核心逻辑的修改
- ❌ ZEGO 账号注册和开通流程
- ❌ 多轮工具链式调用（快速 Agent 单次最多 1 次工具调用）
- ❌ 自定义快速 Agent 的 Prompt 模板（当前由 Plugin 固定构建）
- ❌ 通话录音 / 回放
