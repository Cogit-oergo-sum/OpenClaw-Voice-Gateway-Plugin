## 3. 典型失败模式 (Failure Modes)
1.  **语义粘连 (Semantic Splicing Error)**:
    - 现象：垫词与逻辑回复连读不自然（如“噢早安先生”）。
    - 根因：SLE 首个 Token 吐出时，未感知 SLC 刚才的具体话术。
2.  **长时冷场补偿失效**:
    - 现象：工具执行超过 5s 时，Watchdog 注入的垫词过于机械化（重复“我在呢”）。
    - 根因：缺乏与 OpenClaw 主脑执行过程的实时状态同步。
3.  **盲目执行 (Blind Delegation)**:
    - 现象：在关键信息（如收件人）缺失时，依然触发 `delegate_openclaw`。
    - 根因：SLE 提示词对“执行前置检查”的约束权重不够。

## 4. 后续演进建议 (Priority List)

### P0: 语义缝合与环境感知 (Contextual Smoothing)
- **目标**: 让 SLE 能够“接住” SLC 刚才说的话。
- **动作**: 在 SLE 的 System Prompt 中动态注入前置 SLC 话术镜像，强制对齐语感。

### P1: 动态步进垫词 (Incremental Feedback)
- **目标**: 消除长任务执行期间的机器感。
- **动作**: 将主脑的中间思考状态（Thought Process）作为信号传递给 Watchdog，实现“正在尝试定位地址...”而非“我在呢...”。

### P2: 委派前逻辑门槛 (Pre-Flight Check)
- **目标**: 解决“发空邮件”等低级错误。
- **动作**: 强化 SLE 分类器。若识别到必填参数缺失，强制拦截工具调用，转为直接对话询问。

### P3: TTS 语速节奏动态调节 (Rhythmic TTS)
- **目标**: 让回复听起来更像真人。
- **动作**: 根据回复内容的复杂度，动态下发语速倍率（感性内容加速，技术内容减速）。
